# Pi Study Helper 主要目标

## 产品目标

Pi Study Helper 在 Pi Review Agent 已验证的学习场景基础上重新设计实现，以 Loop Graph SDK 的回路图显式组织学习活动、反馈和资料演进，形成可持续迭代的学习产品。

它同时承担两项同等重要的目标：

1. 提供可用的学习产品：完成材料学习、出题、答题、判题、复盘、归档和会话总结闭环。
2. 验证 Loop Graph SDK：用真实长流程检验图模型、节点执行、状态迁移、上下文投影、完成校验、错误语义和可观测性，并向 SDK 提供带证据的改进建议。

## 项目关系

| 项目 | 在本项目中的角色 |
| --- | --- |
| `pi-study-helper` | 新的主项目，拥有新产品与新契约 |
| `pi-review-agent` | 参考实现，只提供功能清单、交互场景和行为样例 |
| `pi-loop-graph-extension` / `pi-loop-graph-sdk` | 运行时依赖和验证对象 |

本项目不兼容 Pi Review Agent 的代码结构、命令名、Profile 格式或运行数据。可以复用经过重新审查的产品思想，但不建立旧格式迁移器和兼容分支。

## 两个产品里程碑

### 里程碑一：学习会话闭环

- 提供卡片练习、直接练习和章节学习三类场景。
- 以代码节点完成材料展示、答案收集、持久化和题后菜单。
- 以 Agent 节点完成题目生成与语义判题。
- 每题即时归档。
- 正常结束前强制生成并保存学习情况总结。
- 异常中断标记为 `interrupted`，下次允许补生成总结，但不恢复图位置。

### 里程碑二：Profile 构建与修订闭环

- 定义新的 Profile 契约和校验规则。
- 从源资料构建可学习的 Profile。
- 对资料质量执行代码校验与 Agent 语义审查。
- 根据用户反馈修订受影响内容并再次验收。
- 启用前完成结构、来源与质量检查。

## 成功标准

- 关键动作由图和代码保证，不依赖 Agent 猜测何时调用工具。
- 单题默认只有两次必要 Agent Run：生成和判题；会话结束额外一次总结 Run。
- 正常退出不可能绕过总结持久化。
- 图上每条失败路径都有稳定状态、用户可理解说明和诊断证据。
- SDK 反馈能通过最小复现、测试或生命周期 trace 重现。
- 新开发者只读根级工作说明和 docs 即可理解目标、现状与下一步。

## 首期不做

- 旧 Profile 或运行数据导入。
- 多 Agent 通讯、fork/join 或并行图分支。
- 进程终止后的原图恢复。
- Web UI、云同步、多人协作。
- 推理原子图、真题系统和资料分享市场；这些属于后续产品探索，不是本次重构的验收条件。
